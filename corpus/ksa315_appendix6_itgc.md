# 감사기준서 315 보론6 — IT 일반통제(ITGC)의 IT 프로세스별 이해
# (원문 발췌·요약. ITGC 게이트(층1)의 대분류 근거)

감사기준서 315 보론6-1: IT 환경의 각 계층별 IT 일반통제의 성격 — (a) 응용프로그램 계층, (b) 데이터베이스 계층(직접 접근·무단 업데이트 위험 대처), (c) 운영체제 계층(접근관리·자격증명 도용·악성 프로그램 위험 대처), (d) 네트워크 계층(네트워크 분리·원격접속·인증, 웹 기반 응용·제3자 아웃소싱 시 관련).

감사기준서 315 보론6-2(a) 접근 관리 프로세스의 IT 일반통제 예시: 인증(사용자가 본인 자격증명 사용), 권한 부여(직무에 필요한 정보에만 접근·업무분장 가능), 프로비저닝(신규·변경 접근권한 승인), 프로비저닝 해제(퇴사·부서이동 시 접근 삭제), 특별 권한 접근(관리자·강력 사용자 통제), 사용자 접근 검토(주기적 재인증·평가), 보안 설정 통제, 물리적 접근(데이터센터·하드웨어).

감사기준서 315 보론6-2(b) 프로그램 또는 기타 변경 관리 프로세스의 IT 일반통제 예시: 변경 관리 프로세스(변경 설계·프로그래밍·테스트·운영환경 이관), 변경 이관에 대한 업무분장(변경 작성 권한과 운영환경 이관 권한 분리), 시스템 개발·구매·실행(초기 IT 응용프로그램 개발·실행 통제), 데이터 변환(개발·실행·업그레이드 과정의 데이터 변환 통제).

감사기준서 315 보론6-2(c) IT 운영관리 프로세스의 IT 일반통제 예시: 작업 일정관리(재무보고 영향 작업·프로그램의 일정·시작 접근 통제), 작업 모니터링(재무보고 작업의 성공적 실행 모니터링), 백업 및 복구(재무보고 데이터 백업·정전/공격 시 적시 이용·복구), 침입 탐지(IT 환경 취약성·침입 모니터링).

감사기준서 315 A174: IT의 사용으로 인한 위험은 사이버보안과 관련하여 식별될 수 있으며, 내부·외부 서비스제공자 이용(IT 환경 호스팅 아웃소싱, 공유 서비스 센터)에서도 발생할 수 있다. 자동화된 응용통제가 많거나 복잡하고 경영진이 그러한 통제에 더 많이 의존할수록 IT 사용으로 인한 위험이 커진다.

시사점(본 프로젝트): 보론6은 접근·변경·운영이라는 IT 프로세스로 IT 일반통제를 규율하나, AI 고유 리스크(비결정성·모델 드리프트·프롬프트·학습데이터)를 다루지 않는다. 이 공백이 층2 AIGC의 존재 이유다.
